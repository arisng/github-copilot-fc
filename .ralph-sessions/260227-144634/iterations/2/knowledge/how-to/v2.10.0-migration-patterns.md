# v2.10.0 Knowledge Pipeline Refactor — Migration Patterns

## Migrate Librarian Mode References

Replace all references to the old 3-mode pipeline with the new names:

| Find | Replace With |
|------|-------------|
| `MODE: STAGE` (old extraction sense) | `MODE: EXTRACT` |
| `MODE: PROMOTE` (old staging sense) | `MODE: STAGE` |
| `MODE: CURATE` | `MODE: PROMOTE` |

> **Warning:** The names shifted semantically — old STAGE is new EXTRACT, old PROMOTE is new STAGE, old CURATE is new PROMOTE. A naive rename will produce incorrect results. Map by *function*, not by *name*.

## Migrate Signal References

### Remove APPROVE Signal

1. Delete any signal YAML files with `type: APPROVE`.
2. Remove APPROVE from signal type enumerations (e.g., `STEER | PAUSE | ABORT | INFO | APPROVE | SKIP` → `STEER | PAUSE | ABORT | INFO | SKIP`).
3. Remove any signal polling logic that checks for APPROVE signals.
4. Remove APPROVE from signal classification tables and checkpoint documentation.

### Update SKIP Signal Scope

SKIP is now consumed during PROMOTE mode (not CURATE mode):

| Find | Replace With |
|------|-------------|
| `Librarian (CURATE mode)` | `Librarian (PROMOTE mode within KNOWLEDGE_EXTRACTION)` |
| `consumed during the CURATE state` | `consumed during PROMOTE mode within the KNOWLEDGE_EXTRACTION state` |

## Migrate Frontmatter Fields

In all knowledge files under `knowledge/` and `.docs/`:

| Find | Replace With |
|------|-------------|
| `approved: true` | `promoted: true` |
| `approved: false` | `promoted: false` |
| `approved_at:` | `promoted_at:` |

Add new fields if converting from v2.9.0 format:

```yaml
# v2.9.0 format
approved: true
approved_at: 2026-02-27T12:00:00+07:00

# v2.10.0 format
staged: true
staged_at: 2026-02-27T11:00:00+07:00
promoted: true
promoted_at: 2026-02-27T12:00:00+07:00
```

## Migrate Outcome Values

In any orchestrator or librarian contract handling:

| Find | Replace With |
|------|-------------|
| `outcome == "approved"` | `outcome == "promoted"` |
| `outcome == "curated"` | `outcome == "promoted"` |
| `outcome: "approved"` | `outcome: "promoted"` |
| `outcome: "awaiting"` | Remove (no longer a valid outcome) |
| `outcome: "replanning"` | Remove (handled by orchestrator feedback detection) |

## Migrate Progress Tracking

In `progress.md` files:

| Find | Replace With |
|------|-------------|
| `plan-knowledge-approval` | `plan-knowledge-promotion` |

Add the new staging progress item:

```markdown
## Knowledge Progress (Iteration N)
- [ ] plan-knowledge-extraction
- [ ] plan-knowledge-staging       # NEW in v2.10.0
- [ ] plan-knowledge-promotion     # was plan-knowledge-approval
```

## Migrate State Machine References

| Find | Replace With |
|------|-------------|
| `CURATE` (as a state) | Remove — `KNOWLEDGE_EXTRACTION` auto-sequences all three modes |
| `state: CURATE` | Not applicable — no CURATE state exists |
| `KNOWLEDGE_EXTRACTION → CURATE → COMPLETE` | `KNOWLEDGE_EXTRACTION → COMPLETE` (EXTRACT/STAGE/PROMOTE happen within KNOWLEDGE_EXTRACTION) |

## Create Iteration-Scoped Knowledge Directory

For new sessions, the Librarian's Gate 0 preflight auto-creates the structure. For existing sessions missing the directory:

```
iterations/<N>/knowledge/
  tutorials/
  how-to/
  reference/
  explanation/
  index.md
```

The `index.md` manifest tracks all extracted items for the iteration.

## Verify Migration Completeness

After migration, grep for stale references:

```bash
# Check for removed terms
grep -rn "CURATE\|APPROVE\|approved_at\|approved:" agents/ralph-v2/
grep -rn "plan-knowledge-approval" .ralph-sessions/

# Verify new terms are present
grep -rn "EXTRACT\|STAGE\|PROMOTE" agents/ralph-v2/ralph-v2-librarian.agent.md
grep -rn "promoted\|promoted_at" agents/ralph-v2/ralph-v2-librarian.agent.md
```
