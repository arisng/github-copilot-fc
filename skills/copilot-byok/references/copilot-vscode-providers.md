# VS Code BYOK Provider Setup

VS Code uses a **completely different** BYOK mechanism from the CLI. This reference explains how to configure VS Code Chat with the same provider models used by CLI profiles.

> **Key difference**: VS Code ignores `COPILOT_PROVIDER_*` environment variables. Instead, it uses `chatLanguageModels.json` (managed via the `Chat: Manage Language Models` UI command). VS Code supports **multiple providers simultaneously** in its model picker — unlike CLI's single-profile-at-a-time limitation.

---

## Prerequisites

1. A Copilot Pro, Copilot Enterprise, or VS Code extension that supports third-party models
   - Configured via `github.copilot.chat.experimental.modelSwitcher` or `modelSwitcher` for Copilot Chat
  
> **Note**: VS Code's BYOK feature availability depends on your Copilot plan and the VS Code extension version. Not all plan tiers support custom language models. If the `Chat: Manage Language Models` command is unavailable, your plan may not support this feature.

---

## Configuration Entry Points

### Primary UI Path (Recommended)
1. Open Command Palette → **Chat: Manage Language Models**
2. Click **Add Models** → choose a provider type
3. Enter API key and endpoint → models appear in the model picker

### Configuration File
**`%APPDATA%\Code\User\chatLanguageModels.json`**

This file is auto-generated by the UI and can be hand-edited to add/modify model entries. The schema supports these provider vendors:

> **⚠️ CRITICAL — API keys are NOT stored in this file.** When you add a model through the UI, VS Code stores the API key in its **internal secret storage** (credential vault) and writes a secret reference like `"${input:chat.lm.secret.1421b32c}"` into the JSON. **Directly editing `apiKey` with a plain text key or `${ENV_VAR}` placeholder will be silently ignored**, producing "Missing API key" errors. Always use the UI to set or update API keys (gear icon → provider → Update API Key), or delete and re-add the provider through Add Models.

| Vendor | Use Case |
|--------|----------|
| `azure` | Azure OpenAI deployments |
| `openai` | OpenAI API |
| `anthropic` | Anthropic/Messages API |
| `gemini` | Google Gemini |
| `openrouter` | OpenRouter |
| `ollama` | Local Ollama |
| `customendpoint` | Any OpenAI/Anthropic-compatible endpoint (OpenCode Go, Kimi AI, etc.) |

### Workspace-Level Override
`.vscode/settings.json` can include a `chat.languageModels` array scoped to the workspace.

> ⚠️ **Known limitation:** Moonshot's `kimi-k2.7-code` model only accepts `top_p=0.95`, but VS Code always sends `top_p=1.0` and the `chatLanguageModels.json` schema does not support per-model `topP` overrides. Direct `api.moonshot.ai` URLs will fail with a 400 error for this model.

> **Workaround:** Use the local HTTPS proxy described below.

### Kimi AI via local proxy (top_p workaround)

Since VS Code's schema doesn't support `top_p` overrides, route through a local proxy that strips it. Scripts are in the `copilot-byok` skill's `scripts/` folder. Runtime cert data stores at `~/.copilot/moonshot-proxy/`.

**Step 1 — One-time setup** (elevated):
```powershell
pwsh -NoProfile "~\.copilot\skills\copilot-byok\scripts\setup-dns.ps1"
```

Or from the workspace:
```powershell
.\scripts\setup-dns.ps1
```

This adds `127.0.0.1 moonshot.local` to the hosts file and installs a trusted SSL cert.

**Step 2 — Start the proxy** (after every reboot):
```powershell
pwsh -NoProfile "~\.copilot\skills\copilot-byok\scripts\start-proxy.ps1"
```

Or from the workspace:
```powershell
.\scripts\start-proxy.ps1
```

Or add to `.vscode/tasks.json` with `runOn: "folderOpen"` for auto-start.

**Step 3 — Configure `chatLanguageModels.json`** with the proxy URL:
```json
{
  "name": "Moonshot",
  "vendor": "customendpoint",
  "apiKey": "${input:chat.lm.secret.XXXXXXXX}",
  "apiType": "chat-completions",
  "models": [
    {
      "id": "kimi-k2.7-code",
      "name": "Kimi K2.7 Code",
      "url": "https://moonshot.local/v1/chat/completions",
      "toolCalling": true,
      "vision": true,
      "streaming": true,
      "maxInputTokens": 240000,
      "maxOutputTokens": 32768,
      "thinking": true
    }
  ]
}
```

## Per-Agent Model Pinning in VS Code

VS Code supports the `model` field in `.agent.md` frontmatter as a first-class feature. Model names **must** match the `name` field from `chatLanguageModels.json` (not the `id` field).

### String format (single model)

```yaml
---
name: my-agent
description: Specialized reviewer
model: DeepSeek V4 Flash
---
```

### Array format (prioritized fallback)

```yaml
---
name: my-agent
description: Prioritized model fallback
model: ['DeepSeek V4 Pro', 'DeepSeek V4 Flash']
---
```

### Handoff-specific model routing

```yaml
handoffs:
  - label: Quick Research
    agent: explore
    model: DeepSeek V4 Flash
    prompt: Find relevant files
    send: false
  - label: Deep Analysis
    agent: agent
    model: DeepSeek V4 Pro
    prompt: Analyze the findings
```

## Agent-Specific Model Settings (Without Custom Agent Files)

VS Code provides settings for specific agents' default models. These can be set in `.vscode/settings.json`:

```json
{
  "chat.planAgent.defaultModel": "DeepSeek V4 Pro",
  "chat.planAgent.useCurrentModel": true,
  "github.copilot.chat.fixAgent.model": "Kimi K2.7 Code",
  "github.copilot.chat.askAgent.model": "DeepSeek V4 Flash",
  "github.copilot.chat.exploreAgent.model": "DeepSeek V4 Flash",
  "chat.utilityModel": "Qwen3.7 Plus",
  "chat.utilitySmallModel": "GLM-5.1"
}
```

Available settings reference:

| Setting | Purpose | Example |
|---------|---------|---------|
| `chat.planAgent.defaultModel` | Default for Plan agent | `"DeepSeek V4 Pro"` |
| `github.copilot.chat.fixAgent.model` | Fix agent | `"Kimi K2.7 Code"` |
| `github.copilot.chat.askAgent.model` | Ask agent | `"DeepSeek V4 Flash"` |
| `github.copilot.chat.exploreAgent.model` | Explore agent | `"DeepSeek V4 Flash"` |
| `chat.utilityModel` | Summaries, titles, settings search | `"Qwen3.7 Plus"` |
| `chat.utilitySmallModel` | Quick utility (commit messages) | `"GLM-5.1"` |

## Quick Start (UI Path)

### Step 1: Add the provider and store API key

1. Open VS Code → Command Palette → **Chat: Manage Language Models**
2. Click **Add Models** → choose **Custom Endpoint**
3. Enter the configuration:
   - **Name**: `OpenCode Go (OpenAI)`
   - **API Key**: paste your actual raw key (VS Code stores it in secret storage)
   - **API Type**: `Chat Completions`
4. VS Code opens `chatLanguageModels.json` — **do not edit the `apiKey` field**

### Step 2: Add all models to the JSON

Paste the full model list from the [OpenCode Go OpenAI](#opencode-go--openai-compatible-models-deepseek-kimi-glm-mimo) section above into the `models` array. Save the file.

### Step 3: Repeat for Anthropic models

Repeat Step 1 but choose **API Type: Messages** and name it `OpenCode Go (Anthropic)`. Add the Anthropic-compatible models from the [OpenCode Go Anthropic](#opencode-go--anthropic-compatible-models-minimax-qwen) section.

### Step 4: Reload VS Code

After editing `chatLanguageModels.json`, **reload VS Code** (Reload Window command) for changes to take effect.

All configured models appear in the chat model picker dropdown simultaneously.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Model not in picker | `toolCalling` not set to `true` | Copilot Chat requires tool calling — set `"toolCalling": true` |
| 401 errors | Wrong API type for model family | Use `chat-completions` for OpenAI-compatible, `messages` for Anthropic-compatible |
| Context-limit errors | `maxInputTokens` too high for model | Reduce to match the model's actual context window |
| Model picker empty | No models configured with `toolCalling: true` | All Copilot Chat models must support tool calling |
| "Language Models" command missing | Plan tier doesn't support BYOK | Check Copilot plan supports third-party models |
| **"Missing API key" / "Please try again"** | API key was hand-edited into JSON directly, not stored via UI | Use gear icon → provider → **Update API Key** in Language Models editor, or delete + re-add via **Add Models → Custom Endpoint** |
